Setting Value Setting Description
TAF/MOS Hour to Display Time in Hours of the TAF/MOS data to display. Range is 1 to 24. Used when rotary switch is NOT installed.
Probability to Display MOS Data For MOS Data (US Only). Probability in % that a particular reported weather will occur at the airport. MOS data uses a different model which requires the builder to set a probability level in which to display the weather on the map. i.e. 50 would equal a 50% chance. 75 would be a 75% chance.
Display Binary or Gradient Colors For Heat Map. Highlights which airports have been landed at. The LEDs can display all the visited airports in Red (Binary) or depending on the number of landings, can display a gradient of color from Cool Blue for few visits, to Hot Red for lots of visits.
For Heat Map. Using Green to denote home airport will make the home airport stand out from the rest of the airports visited.
For Heat Map. This setting will fade the Green of the home airport in and out to make it stand out further from the rest of the airports.
For Heat Map. Delay time to change how slowly the home airport LED fades in and out. .01 - .05 is typical. Range is 0 - 3 with a step value of .001
When the map updates the FAA weather, a transitional wipe can be used to fill the time and let people know the update is occuring. Various wipes are available, some utilize the airport's Lat/Lon to create effects. The spacing and volume of airports on a map will dictate how these wipes will look. So play with them to find the one(s) that work best for the map.
